# Product

## Register

product

## Platform

web

## Users

Solo developers who run several AI coding agents daily — Claude, Codex, OpenCode, Kimi, and plain shells — across multiple project folders. They live in the keyboard, keep many agents running in parallel, and open Terra Swarm at the start of a session and leave it open all day. Their job to be done: direct and monitor a fleet of agents without thinking about the windows those agents live in.

## Product Purpose

Terra Swarm is mission control for AI coding agents: every agent runs as a real PTY in one native window, organized into a workspace per project and a resizable terminal grid per workspace, with live context tracking, notifications, voice input, and agent-native session resume. It exists because juggling terminal windows and tabs is the tax on working with multiple agents. Success looks like the user forgetting window management exists — agents just run side by side, and attention goes to the work, not the chrome.

## Positioning

Mission control for AI coding agents. Not one feature but the combination: every agent side by side as a real PTY in a single native window, monitored at a glance, and resumable exactly where it left off. Every screen reinforces that this is the one window where a fleet of agents is run, watched, and picked back up.

## Brand Personality

Quiet, precise, fast. The reference is Warp: a modern terminal done seriously — craft and performance as the personality, nothing decorative that doesn't carry state.

## Anti-references

IDE chrome overload: every pixel fighting the terminals for attention. Toolbars, panels, and ornamentation that compete with agent output instead of serving it.

## Design Principles

- The terminal grid is the product. Every other element earns its pixels by carrying state — notifications, context usage, git branch — or it isn't there. Chrome recedes; agent output owns the window.
- Forget the window manager exists. Spawn, switch, resize, resume: if the user has to think about the container, the design failed. Interactions should feel instantaneous and keyboard-first.
- Say it once, say it exactly. Quiet, precise, fast applies to copy, iconography, and motion alike — no redundancy, no ornament, no animation that delays the work.
- Take the terminal as seriously as Warp does. Real PTYs, native-feeling performance, and terminal-grade typography are the craft bar, not an afterthought wrapped in a webview.
- One language, every desktop. Cross-platform neutral: no mac-only conventions in the visual design, with shortcuts adapted per OS rather than a design that drifts per OS.

## Accessibility & Inclusion

WCAG AA as the floor, with real care beyond it for reduced motion and low-vision contrast — every animation has a reduced-motion alternative, and text contrast is checked against the dark surfaces rather than assumed.
